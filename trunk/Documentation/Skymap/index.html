Skymap

A medium accuracy star chart/sky map/ephemeris program for matlab.

What does it provide?

This program provides an easy to use sky map with medium accuracy. That is it plots the stars as they would be seen in the sky from a given point on the earth at any given time. What is meant by ``medium accuracy'' is that the higher order corrections (nutation, proper motion, parallax) for calculating the azimuth and zenith position of the stars is not taken into account.
Due to lack of marked needs, the program currently only plot he stars from the Bright Star Catalog (BSC) and no planets or other bright celestial objects.

In addition stellar spectra from the Pulkovo Spectrophotometric Catalog can be extracted - for the 600-900 stars that have their absolute spectral intensities tabulated.

The features are:

  1. Rect ascension/Declination or Azimuth/Zenith grid
  2. slide rulers for field-of-view and THREE rotation angles of the f-o-v.
  3. Pop-up menu for faintest magnitude
  4. Preference dialog for plot colors and sizes
  5. Easily editable files of observatory names, and positions.
  6. Integrated with the Pulkovo spectroscopic catalog - providing stellar spectra with 5 nm resolution from 300-1100 nm
  7. Integrated with the StarCal camera calibration program.

Installation.

Simple. Really.
  1. Put the tar file to somewhere you want your mat-files, like ~/matlab
  2. tar -xvf Skymap.tar
  3. Add the Skymap directory to your matlab-path.
  4. Done.

Customization

Not very much to customize, except the list of observation sites. This is easily don by modify/add/remove as wanted in the files .data/station.name and .data/stationpos.dat. In stationpos.dat the format is 
  67 50 26.58777 1 20 24 40.4508  1
      
degrees arc-minutes arc-seconds NS-flag  degrees arcmin arcsec EW-flag
Here NS-flag and EW-flag is positive for north and east respectively.

A significantly more cumbersome customization is to add other star catalogs to the program.
The best way to do this is to rewrite the functions related to loading the star catalog, if you need to do this please contact the author for help or consult the source code documentation.
The easier way is possibly just to append additional stars into the files stars/ybs.bjg and stars/ybs.org. This will of course make the somewhat slow loading of the stars even slower.

Run the program

At the matlab prompt any of the following function calls will work: 
 [SkMp] = skymap;
 [SkMp] = skymap(pos,time);
 [SkMp] = skymap(pos,time,SkMp);
for programming reasons it is necessary to literary name the output SkMp. "pos" should be a 2x1 vector with [lat long] in degrees with north and east positive, and time should be a 6x1 vector with format [yyyy mm dd hh mm ss.fos]. With no arguments passed you will be prompted with a figure to select date and time (UT) and observation position. When pressing "Ok" the program will load the star catalog and calculate the positions of the stars. The usage of the GUI is straightforward, the top slide ruler change the field of view the right one change the rotation around the middle of the field-of-view, the left and right rotates the field of view around east-west and north-south. The pop-up menu sets the faintest stars to display. From the "Star" menu you can select Ra/Decl or Az/Ze grid and get information on a specific star. From the "Pos/time" you can request the current values or change to a new set of parameters. From the "File" menu you can print the sky map, close the session, and set various preferences.
