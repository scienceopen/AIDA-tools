Rotations and their matrices

When it comes to rotations there are a plethora of options. Which rotations to choose, which order to perform them, from where to start the rotation, which is the positive direction...

However, as long as we determine three rotations that are not degenerate we can choose pretty much as we like - as long as we don't mix them up. Starcal/Camera and the other toolboxes built around them uses rotations provided by these two functions:

  1. camera_rot.m
    Function that determines the rotation matrix to the the camera coordinate system (e3 as the optical axis). This function uses the full 3 angle Tait-Bryant (Gyro-mount) rotations.
  2. camera_base.m
    Function that calculates the unit vectors of the rotated camera system.
